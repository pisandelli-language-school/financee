## 2025-07-15 - Unsafe error reflection & missing security headers
**Vulnerability:** The OAuth confirmation page (`confirm.vue`) reflected raw `error` or `error_description` query parameters directly into the UI. In addition, the application lacked default security headers.
**Learning:** Nuxt does not add standard security headers like `X-Frame-Options` or `X-Content-Type-Options` by default. Reflecting raw URL query parameters directly into the UI without sanitization or mapping introduces a risk of content spoofing/social engineering, especially during sensitive operations like authentication.
**Prevention:** Avoid reflecting raw error parameters from the URL directly to the user. Instead, check for the presence of an error, log the raw parameters for debugging, and show a safe, generic error message. Always configure `routeRules` in `nuxt.config.ts` to include standard HTTP security headers (e.g., `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, and `Strict-Transport-Security`).
