## 2025-07-16 - Prevent Authorization Bypass in Dynamic Backoffice Routes
**Vulnerability:** The dynamic backoffice endpoints (`server/api/backoffice/[section].ts` and `server/api/backoffice/[section]/[id].ts`) were using `requireSupabaseUser`, which only validates authentication. This allowed any authenticated user to bypass RBAC and access, create, modify, or delete resources in any section.
**Learning:** In Nuxt/Nitro applications, dynamic route parameters (like `[section]`) need strict validation against an allowlist and specific RBAC permission checks, especially when mapping to underlying models. Relying solely on a base authentication check for multi-tenant or varied-role applications exposes high-priority endpoints to privilege escalation.
**Prevention:** Always pair dynamic route parameter access with explicit permission mappings (`requirePermission(event, \`\${moduleName}.\${action}\`)`) and use a strict allowlist to validate inputs instead of passing unvalidated parameters directly to backend utilities.
