## 2025-02-14 - Fix Authorization Bypass via Incomplete RBAC on Backoffice Endpoints
**Vulnerability:** The backoffice generic API endpoints (`/api/backoffice/[section]` and `/api/backoffice/[section]/[id]`) were protected using `requireSupabaseUser(event)`, which only verified authentication and did not enforce proper Role-Based Access Control (RBAC). This resulted in an authorization bypass where any authenticated user could read or modify sensitive resources (e.g., categories, contacts, etc.) regardless of their assigned permissions.
**Learning:** Depending solely on general authentication validation (`requireSupabaseUser`) for administrative or specific CRUD routes leaves applications vulnerable to horizontal and vertical privilege escalation. Endpoints performing sensitive operations must map directly to RBAC permissions.
**Prevention:** Always enforce authorization at the endpoint level by mapping requested actions and modules directly to system permissions (e.g., using `requirePermission`). Validate the module access dynamically if generic routes are used, ensuring that every dynamic path still aligns with an established permission schema.
