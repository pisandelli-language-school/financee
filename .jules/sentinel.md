## 2025-02-27 - Authorization Bypass in Dynamic Backoffice Routes
 **Vulnerability:** The dynamic backoffice endpoints (`server/api/backoffice/[section].ts` and `[section]/[id].ts`) were protected only with `requireSupabaseUser`. This only validated that the user was authenticated, completely bypassing the RBAC (Role-Based Access Control) system and allowing any authenticated user to interact with all backoffice endpoints (escalation of privileges).
 **Learning:** In a single-tenant architecture with RBAC, relying on authentication checks without authorization logic is a critical vulnerability. Dynamic routes that abstract multiple modules need to dynamically enforce RBAC checks matching the route parameter and HTTP action being performed.
 **Prevention:** Always enforce authorization using `requirePermission` (or similar RBAC mechanisms) in API handlers, rather than just `requireSupabaseUser`. Ensure dynamic routes correctly map route parameters to their respective permissions based on the performed CRUD action.
